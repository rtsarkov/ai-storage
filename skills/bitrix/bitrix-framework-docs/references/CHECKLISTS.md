# Checklists

## Общий gate

- [ ] Решение находится в `/local/`, а не в ядре `/bitrix/`.
- [ ] Локальная версия ядра поддерживает использованные классы и методы.
- [ ] Существующие проектные паттерны соблюдены.
- [ ] Есть проверка прав на уровне action/service.
- [ ] Данные пользователя не попадают в SQL без `SqlHelper`/`SqlExpression`/ORM.
- [ ] Ошибки возвращаются в формате, принятом в проекте.
- [ ] `php -l` или более сильная проверка выполнена для измененных PHP-файлов.

## Controller gate

- [ ] `*Action` имеет явные типы параметров и результата, если стиль проекта это допускает.
- [ ] `configureActions()` не отключает default filters без причины.
- [ ] Write-action защищен authentication + CSRF + domain permission check.
- [ ] Read-only action с долгой обработкой закрывает сессию, если это безопасно.
- [ ] Response type выбран осознанно: array/json/file/component/html/redirect.
- [ ] Frontend payload и response shape документированы для вызывающей стороны.

## ORM/query gate

- [ ] `select`, `filter`, `order`, `limit` не вытягивают лишние поля.
- [ ] Нет per-row запросов в списке.
- [ ] Для 1:N/N:M проверены `LIMIT` и декартово произведение.
- [ ] Runtime relations не ломают `fetchCollection()` и object hydration.
- [ ] Кеш ORM включен/отключен осознанно.
- [ ] Индексы соответствуют фильтрам и сортировкам.

## Cache gate

- [ ] Ключ кеша включает все влияющие параметры.
- [ ] Есть понятный TTL или managed invalidation.
- [ ] При изменении данных очищается нужный cache layer.
- [ ] Кеш не смешивает данные разных пользователей, ролей, сайтов или языков.
- [ ] Для статического кеша есть reset при тестах или изменении состояния в рамках запроса.

## Module/event gate

- [ ] `install/index.php` не содержит runtime-бизнес-логики.
- [ ] `include.php` подключает только нужную автозагрузку/совместимые функции.
- [ ] Event handlers регистрируются при установке и снимаются при удалении.
- [ ] Legacy events обработаны с учетом сигнатуры старого API.
- [ ] Миграции/установка БД обратимы или явно описывают необратимые шаги.

## JS extension gate

- [ ] Entry point указан в `bundle.config.js`.
- [ ] CSS импортируется из JS entrypoint или явно указан в `config.php`.
- [ ] `rel` содержит зависимости, если сборщик их не добавляет автоматически.
- [ ] PHP подключает extension через `Bitrix\Main\UI\Extension::load()`.
- [ ] Lazy-load используется только там, где пользовательский сценарий это оправдывает.
